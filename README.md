<h2>About Me</h2>

<p>
I am a Computer Engineer and iOS Developer with a strong focus on building clean, scalable and testable iOS applications.
I have hands-on experience developing production-ready apps using <b>Swift, SwiftUI and UIKit</b>, while applying modern architectural patterns such as <b>MVVM, VIPER and Clean Architecture</b>.
</p>

<p>
I enjoy working with <b>Combine</b> and <b>Swift Concurrency</b> to design reactive and asynchronous data flows.
My expertise includes integrating RESTful APIs, managing local persistence with <b>Core Data and SwiftData</b> and maintaining a modular and extensible codebase. Quality and maintainability are core priorities for me. I actively apply <b>Protocol-Oriented Programming</b>, write <b>Unit & UI tests</b> and support projects with <b>CI/CD pipelines</b> to ensure reliability and long-term scalability.
</p>

      <h2>Weather App</h2>
      <p>
        A Swift UIKit-based iOS weather application built using <b>MVVM architecture</b>.
        It delivers real-time weather data through a custom <b>NetworkManager powered by Alamofire</b>,
        including hourly and 7-day forecasts.
      </p>
      <p>
        The app features dynamic weather-based GIF backgrounds using <b>SwiftyGif</b>,
        optimized image loading with <b>Kingfisher</b>, and animated splash screens powered by <b>Lottie</b>.
      </p>
      <p>
        Users can save favorite locations with <b>CoreData</b> persistence, visualize them via <b>MapKit annotations</b>,
        and interact with an NLP-powered AI Assistant for conversational weather queries.
      </p>
      <p>
        Additional features include custom URL-scheme deep linking and comprehensive unit testing
        for both ViewModel and networking layers.
      </p>
